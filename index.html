一、linux操作命令（一）

ls命令

ls 选项 目录名

-a：列出所有文件，包括以.开头的文件

-l：文件权限，所有者，文件大小，时间等信息，还有文件名都列出来

-h：-human-readable 以容易理解的方式列出文件大小，从小到大或者从大到小

-t：以文件修改时间排序

  1. d表示目录，-表示文件，文件和目录的颜色也不一样
  2. ls -l d*开头为d的文件
  3. ls -alh /home 列出home目录下所有的适合理解的文件大小

cd命令

cd 目录名

pwd 显示当前所在目录

cd / 从当前目录进入系统根目录

cd ..从当前目录进入父目录

cd ~进入当前主目录（区别于根目录）

cd -进入上次所在目录

pwd命令

pwd -P

pwd -L

二、linux操作命令（二）

mkdir命令

mkdir 选项 目录

./是当前目录，/是根目录，../是父目录

-m – 设定模式（权限）

-p – 可以是一个路径名称，一次性建立多个目录

-v – 每次创建新目录都显示信息

  1. mkdir -m 777 zhou 创建777权限的zhou文件
  2. mkdir -p ./zhou/test创建多个目录
  3. mkdir -vp ./zhou/test返回创建信息

rm命令

rm 选项 文件或目录

-f force 忽略不存在的文件，从不给出提示

-i interactive进行交互式删除

-r recursive 将列出的目录和子目录递归的删除

-v 详细显示删除步骤

  1. sudo touch 1.log创建文件1.log
  2. rm删除有询问，rm -f删除不提示
  3. mkdir创建文件目录
  4. -v显示具体情况，-r递归删除

mv命令

mv 选项 源文件或目录 目标文件或目录

-b back：如果覆盖文件，覆盖前先备份

-f force：如果目标文件存在，直接覆盖不询问

-i interactive：如果目标文件存在，询问是否覆盖

-t target：移动多个源文件到同一个目录，目标目录在前

mv old.log new.log 重命名

mv old.log test(已存在目录) 移动到test目录下

mv old.log test(不存在目录) 重命名为test

mv -i old.log test(已存在old.log) 覆盖前询问是否覆盖

cp命令

cat命令

cat [选项] [文件]

-b：对非空输出行编号

-n：对输出多有行进行编号

把shiyanlou.log中的内容输入到zhou.log中，tac为反向从屏幕中显示

nl命令

nl [选项] [文件] numbers of lines

-b：指定行号指定的方式

-b a：不论是否空行，都列出行号

-b t：如果有空行，那一行不列出行号（默认）

-n：列出行号标识的方法

-n ln：行号在屏幕最左方显示

-n rn：行号在自己栏位的最右方表示，不加0

-n rz：行号在自己栏位的最右方表示，加0

-w：行号栏位占用的位数

三、linux操作命令（三）

move命令

more [选项] 文件

+n：从第n行开始显示

-n：定义屏幕大小为n行

+/pattern：搜索该字串，从该字串前两行开始显示

-c：从顶部清屏，然后显示

=：输出当前行号

q：退出more

空格键：向下滚动一屏

b：返回上一屏

ll /目录 | more -n 显示目录信息

less命令

less [选项] 文件

head命令

head [选项] [文件]

-q：隐藏文件名

-v：显示文件名

-c<字节> ：显示字节数

-n<行数> ：显示行数

tail命令

tail [选项] [文件]

-f：循环读取，可以刷新读取的内容

-q：不显示处理信息

-v：显示详细的处理信息

-c<字节>：显示的字节数

-n<行数>：显示的行数

ping www.shiyanlou.com >> shiyanlou.log & ping远程主机，并将信息追加到shiyanlou.log文件中，&的命令是将这条命令放在后台执行，这样shiyanlou.log文件就会一直有内容增加。

linux下执行ping命令会一直执行，必须手动停止才行；而windows下执行ping命令时，默认发送四个请求后会自动停止。

jobs命令可以查看正在后台运行的任务；kill命令可以杀死一个任务，但要使用任务的pid；任务的pid可以通过ps命令查看获得；kill -9 pid可以杀死后台进程。

四、linux操作命令（四）

which

which 可执行文件名称

which命令的作用是在PATH变量指定的路径中搜索可执行文件的所在位置，一般来确认系统中是否安装了指定的软件。

cd在这里是一个shell内置命令，但是在某些环境下cd不是一个内置函数而是一个shell函数

whereis

whereis [选项] 文件

-b：定位可执行文件；-B：指定搜索可执行文件的路径

-m：定位帮助文件；-M：指定搜索帮助文件的路径

-s：定位源代码文件；-S：指定搜索源代码文件的路径

-u：搜索默认路径下除可执行文件、定位帮助文件、源代码文件以外的其他文件

默认情况下，whereis 是从由匹配符所定义的硬编码路径来寻找文件的。但如果你想的话，你可以用命令行选项来限制搜索。例如，如果你只想在 /usr/bin 寻找二进制文件，你可以用 -B 这一选项来实现。

whereis-B /usr/bin/-f cp

注意：使用这种方式时可以给出多个路径。使用 -f 这一选项来明确分隔目录列表和要搜索的文件名。

locate命令

locate [选项] [搜索字符串]

-q：安静模式，不会显示任何错误信息

-n：至多显示n个输出

-r：使用正则表达式做寻找的条件

-V：显示版本信息

使用特殊子元*和？来指定查找的样本

先安装并更新数据库，再匹配搜索

六、linux操作命令（六）

wc命令

wc [选项] [文件] word count的缩写

-c：统计字节数

-l：统计行数

-m：统计字符数，不能与-c标志一起使用

-w：统计字数，一个字被定义为由空白、跳格或换行字符分隔的字符串

-L：打印最长行的长度

不打印文件名使用命令cat c.txt | wc -o

统计/bin目录下命令的个数：ls /bin | wc -l

grep命令

grep [选项] pattern [file]

-c：计算找到“搜索字符串”（即pattern）的次数

-i：忽略大小写的不同，大小写视为相同

-n：输出行号

-v：反向选择，打印不匹配的行

-r：递归搜索

-color=auto：将找到的关键词部分加上颜色显示

grep “root” /etc/passwd –color=auto

cat /etc/passwd | grep “root” –color=auto

-v反向，-r递归查找

正则表达式与grep命令

正则表达式 描述 示例
^ 行开始标记 ^shiyan匹配以shiyan起始的行
$ 行尾标记 $shiyan匹配以shiyan结尾的行
. 匹配任意一个字符 a.c匹配abc，aac但不匹配abbc
[] 匹配包含在[字符]中的任意一个字符 ab[cd]匹配abc或者acd
[^] 匹配除[ ^字符 ]以外的任意一个字符 1[ ^01]匹配12、13，但不匹配10、11
[-] 匹配[]指定范围内的任意一个字符 [1-5]匹配1-5的任意一个数字
{n} 匹配之前的项n次 [0-9]{2}匹配一个两位数，相当于[0-9] [0-9]
{n,} 匹配之前的项至少n次 [0-9]{2,}匹配任意一个两位或者更多位的数字
{n,m} 指定之前的项需要匹配的最小和最大的次数 [0-9]{2,5}匹配从两位数到五位数之间的任意一个数字
? 匹配之前的项1次或者0次 shiy?an匹配shiyan或者shian
* 匹配之前的项0次或者多次 shiy*an匹配shian、shiyan、shiyyan
+ 匹配之前的项1次或者多次 shiy+an匹配shiyan、shiyyan
() 创建一个用于匹配的字串 ma(in)?匹配main或者ma
| 匹配|两边的任意一项 Dec(1st|2nd)匹配
\ 将上面的特殊字符进行转义 a \ +b匹配a+b

例一：利用linux系统自带的字典查找一个五个字母的单词，第三个字母为j，最后一个字母为r，/usr/share/dict目录下存放字典文件words

例二：验证固定电话：固定电话格式基本都是带有0的区号+连接符‘-’+电话号码，另外还可能有分机号，区号有3、4为，电话号码有7、8位，分机号一般都是3-4位数字

正则表达式为：”^0[0-9]{2,3}-[0-9]{7,8}(-[0-9]{3,4})?$”

必须要有-E，没有-E第二个命令查询不到结果

例一没有-E却查询成功是因为grep把.当作shell的通配符而不是正则表达式的元字符。
